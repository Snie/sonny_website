# Reporting a Vulnerability

I harden this private portfolio website of myself regularly on any new feature, and I have codeQL, dependabot, so as secrets scanning enabled. However,
I may miss some vulnerability or CSP standards as I am not a security researcher.
So I Invite you to ethically try to pwn my website and report any finding to me,
as I am always eager to learn, especially important and crucial security concepts that benefit every computer scientist.
You guys are doing a great work out there, and nowadays security is more important than ever!

Please use [GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/how-tos/report-and-fix-vulnerabilities/privately-reporting-a-security-vulnerability) to report me any vulnerability you find here.  

## How to report

Check the link above to read GitHub official documentation, anyway here a quick guide for reporting:

`Security tab --> "Report a vulnerability"`

## Scope

### In scope

- The Next.js application code
- GitHub actions workflows
- Static assets in /public
- Node packages and next configurations
- Testing and testing configuration
- Claude code configuration
- Spec kit configurations and specifications themselves (.specify and specs)

### Out of scope

Nothing :) Threats hide where people doesn't expect them to be ;)
